# Перенос на российский контур (чеклист)

MVP-ориентированный список шагов без привязки к конкретному облаку.

1. **База данных**  
   - Снять дамп PostgreSQL (`pg_dump`).  
   - Восстановить в целевом кластере.  
   - Обновить `DATABASE_URL`.

2. **Объектное хранилище**  
   - Синхронизировать bucket с ключами `companies/...`, `tenders/...`.  
   - Обновить `S3_ENDPOINT`, `S3_REGION`, `S3_BUCKET_UPLOADS`, ключи доступа, при необходимости `S3_FORCE_PATH_STYLE`.

3. **Redis**  
   - Поднять managed Redis или свой инстанс.  
   - Обновить `REDIS_URL`. Очереди BullMQ можно очистить при переносе (задачи не критичны как долговременные данные).

4. **AI-gateway**  
   - Развернуть `apps/ai-gateway` в закрытом контуре.  
   - Обновить `AI_GATEWAY_BASE_URL` в web/worker.  
   - Выдать новый `AI_GATEWAY_API_KEY`.  
   - Настроить доступ к выбранной LLM API (ключи только в ai-gateway).

5. **Web / Worker**  
   - Собрать образы или задеплоить на VM/k8s.  
   - Пробросить все переменные из корневого `.env` (см. `docs/env.md`) + домен в `NEXT_PUBLIC_APP_URL` при необходимости.

6. **DNS и TLS**  
   - Настроить сертификаты для публичного домена приложения.

7. **Проверка**  
   - Регистрация → компания → закупка → файл → worker → AI-разбор → черновик → экспорт ZIP.
